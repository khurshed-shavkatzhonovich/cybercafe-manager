# CyberCafe Manager — Руководство по системе лицензирования

## Архитектура

Система использует RSA-2048 асимметричную криптографию:

- **Приватный ключ** (`scripts/license-keys/private.pem`) — только у разработчика, используется для подписи лицензий
- **Публичный ключ** (встроен в `src/main/license.js`) — находится внутри приложения, используется для проверки

Клиент не может сгенерировать себе ключ, так как у него нет приватного ключа.

---

## Первоначальная настройка (один раз)

Ключи уже сгенерированы и настроены. Не нужно делать ничего дополнительно.

**ВАЖНО**: Файл `scripts/license-keys/private.pem` — строго секретный.
- Никогда не коммить его в git (уже добавлен в .gitignore)
- Сделай резервную копию в надёжном месте (флешка, зашифрованное облако)
- Если потеряешь — нужно будет перевыпустить ключи для всех клиентов

## Выдача лицензии клиенту

### Шаг 1 — Получить Machine ID клиента

Попроси клиента открыть приложение и перейти в:
**Настройки → Лицензия → Идентификатор компьютера**

Клиент нажимает кнопку копирования и отправляет тебе строку вида:
```
a1b2c3d4e5f6789012345678901234567890abcd
```

### Шаг 2 — Сгенерировать ключ

На своём компьютере (там, где хранится `private.pem`):

```bash
node scripts/generate-license.js
```

Скрипт задаёт вопросы:
1. **Machine ID клиента** — вставь строку, полученную от клиента
2. **Название клуба** — например `GameZone` или `Нео Клаб`
3. **Тип лицензии**:
   - `1` — Годовая (365 дней)
   - `2` — Двухлетняя (730 дней)
   - `3` — Бессрочная (до 2099 года)

### Шаг 3 — Отправить ключ

Скрипт выведет длинную строку — лицензионный ключ. Отправь его клиенту любым удобным способом (WhatsApp, Telegram, email).

Ключ выглядит примерно так:
```
eyJtIjoiZDAxMmE2NGJiM2FmNzA0ZTJjN2NjMjdkY2ZhMDg3OWE0YzNlMTY3NTRmZTY3OTg1YWZjYjE2YTFlYjJlOTFhZCIsImMiOiJUZXN0Q2x1YiIsImkiOiIyMDI2LTA2LTA3IiwiZSI6IjIwMjctMDYtMDciLCJ0IjoiYW5udWFsIn0.ABC123...
```

### Шаг 4 — Клиент активирует

Клиент идёт в **Настройки → Лицензия**, вставляет ключ и нажимает **Активировать**.

## Продление лицензии

Процедура та же — просто сгенерируй новый ключ с новым сроком действия. Старый ключ перестаёт работать при следующей проверке.

---

## Режимы работы лицензии

| Статус | Дней до конца | Что происходит |
|--------|--------------|----------------|
| `active` | > 14 | Всё работает нормально |
| `grace` | 1–14 | Предупреждение в шапке и в боковом меню, всё работает |
| `readonly` | < 0 | Режим только чтения: можно смотреть данные и закрывать открытые счета, нельзя создавать новые |

---

## Защита от манипуляций с датой

При каждом запуске приложение сохраняет дату последнего запуска в зашифрованном файле `.meta` (в папке данных пользователя).

**Срабатывает предупреждение** (красная полоса в шапке) если:
- Системная дата более чем на 2 дня **раньше** даты последнего запуска
- Системная дата **раньше** даты первого запуска приложения

Предупреждение не блокирует работу — только информирует пользователя и записывается в лог.

---

## Журнал лицензий (твой учёт)

Каждый выданный ключ записывается в:
```
scripts/license-keys/issued-licenses.log
```

Формат записи:
```
[2026-06-07T10:30:00.000Z] Club="GameZone" Type=annual MachineID=a1b2c3d4... Expires=2027-06-07
```

## Пересоздание ключей (АВАРИЙНАЯ СИТУАЦИЯ)

Делается только если приватный ключ был скомпрометирован:

```bash
# Удали старые ключи
del scripts\license-keys\private.pem
del scripts\license-keys\public.pem

# Сгенерируй новые
node scripts/generate-keys.js

# Скопируй новый PUBLIC KEY в src/main/license.js (поле PUBLIC_KEY)

# Пересобери и выпусти новую версию приложения

# Перевыпусти ключи ВСЕМ существующим клиентам
```

**ВНИМАНИЕ**: После замены ключей все старые лицензии перестают работать!

## Файловая структура лицензирования

```
scripts/
├── generate-keys.js          ← Генерация RSA ключевой пары (выполнить один раз)
├── generate-license.js       ← Генерация лицензии для клиента
└── license-keys/
    ├── private.pem           ← СЕКРЕТ — только у тебя, не в git!
    ├── public.pem            ← Публичный ключ (дублирует то, что в license.js)
    └── issued-licenses.log   ← Лог выданных лицензий

src/main/
└── license.js                ← Менеджер лицензий (публичный ключ встроен)

%APPDATA%\cybercafe-manager\  ← Папка данных пользователя
├── .lic                      ← Файл лицензии (лицензионный ключ)
└── .meta                     ← Метаданные дат запуска (base64, защита от rollback)
```
